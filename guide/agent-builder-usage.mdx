# Step 5: 에이전트 빌더 실습

## 1. AI 에이전트 자동 생성

"5분 만에 나만의 AI 비서를" 화면에서 원하는 에이전트를 자연어로 설명하면 AI가 에이전트 정보와 구성 블록을 자동 세팅합니다.

<Steps>
  <Step title="생성 탭 선택">
    상단 탭에서 **[생성]** 을 클릭합니다.
  </Step>

  <Step title="자연어 설명 입력">
    "만들고 싶은 에이전트를 자유롭게 입력하세요" 입력란에 원하는 기능을 자연어로 설명하거나, 하단의 추천 템플릿(챗봇 · 창업 코치 · 자소서 코치 · 진로 탐색기 등)을 선택합니다.

    <Frame caption="에이전트 생성 — 자연어 입력과 추천 템플릿">
      <img src="/images/에이전트 빌더_생성.png" alt="에이전트 생성 화면" />
    </Frame>
  </Step>

  <Step title="AI 자동 구성">
    전송 버튼(↑)을 클릭하면 AI가 **에이전트 정보(이름·설명·모델·시스템 프롬프트)** 와 **필요한 구성 블록(KNOWLEDGE · TOOLBOX · SKILLS 등)** 을 자동으로 세팅합니다.
  </Step>

  <Step title="에이전트 편집 화면으로 이동">
    생성 완료 후 에이전트 편집 화면(Workflow 뷰)으로 자동 이동합니다. 좌측 대화 패널에서 테스트하고, 우측에서 각 블록을 세부 조정합니다.

    <Frame caption="에이전트 편집 화면 — 좌측 대화 / 우측 Workflow 뷰">
      <img src="/images/에이전트 빌더 화면.png" alt="에이전트 편집 화면" />
    </Frame>
  </Step>
</Steps>

<Tip>
자연어 자동 생성이 자동 세팅하는 범위는 **시스템 프롬프트 + TOOLBOX** 까지입니다. SCHEDULE · CHANNELS · FEW-SHOT · SUB-AGENTS 는 수동으로 추가해야 합니다.
</Tip>

## 2. 에이전트 화면 구성

에이전트 편집 화면은 좌우로 분할됩니다.

### 좌측 대화 패널

<Tabs>
  <Tab title="에이전트 대화">
    설정한 에이전트와 실시간 테스트 대화. 연결된 KNOWLEDGE · TOOLBOX · SKILLS · SUB-AGENTS 를 활용해 답변을 생성합니다.
  </Tab>

  <Tab title="AI 수정 도우미">
    자연어로 에이전트 수정을 요청합니다. **기존의 워크플로우 검증 · AI 품질 분석 · 수정 요청 기능이 이 탭으로 통합**되었습니다.

    예: "응답을 더 간결하게 해줘" / "서명은 KNOWLEDGE 파일의 B 섹션 값만 사용하도록 해줘" → AI가 PROMPT · 블록을 자동 수정합니다.
  </Tab>
</Tabs>

### 우측 편집 영역

<Tabs>
  <Tab title="Workflow 뷰">
    에이전트 정보 블록 + 8종 구성 블록을 시각적으로 배치한 기본 뷰. 각 블록의 `+ Add` 또는 `Edit` 버튼으로 항목을 관리합니다.

    <Frame caption="Workflow 뷰 — 8종 구성 블록">
      <img src="/images/에이전트 수정노드.png" alt="Workflow 뷰" />
    </Frame>
  </Tab>

  <Tab title="Folder 뷰">
    에이전트에 연결된 자산(지식베이스 · 스킬 · 설정 파일 등)을 **폴더 구조**로 조회합니다.
  </Tab>
</Tabs>

## 3. 에이전트 정보 설정

중앙 **에이전트 정보** 블록 우측 상단의 **`Edit`** 버튼을 클릭하면 **[에이전트 설정]** 모달이 열립니다.

<Frame caption="에이전트 설정 — 이름·설명·모델·시스템 프롬프트">
  <img src="/images/프롬프트 Edit 클릭시.png" alt="에이전트 설정 모달" />
</Frame>

| 항목 | 설명 |
|------|------|
| **에이전트 이름** | 식별용 이름 (예: "신입사원 온보딩 네비게이터") |
| **설명** | 1~2줄 목적 요약 |
| **모델** | gpt-4.1 등 LLM 모델 선택 |
| **프롬프트 불러오기** | 저장된 시스템 프롬프트를 선택하여 가져옵니다 |
| **시스템 프롬프트 본문** | 역할 · 규칙 · 출력 형식을 자유 편집 |

<Tip>
자주 쓰는 프롬프트는 별도 저장해 **[프롬프트 불러오기]** 로 재사용하세요. 팀·수업에서 공유 가능한 프롬프트 자산이 됩니다.
</Tip>

## 4. 구성 블록 — 8종 설정

모든 블록은 우측 상단 **`+ Add`** 버튼으로 항목을 추가합니다.

### 4-1. KNOWLEDGE — 지식베이스 연결

<Steps>
  <Step title="+ Add 클릭">
    KNOWLEDGE 블록의 **`+ Add`** 버튼을 클릭합니다.
  </Step>

  <Step title="파일 업로드 또는 기존 선택">
    파일을 드래그하거나 클릭하여 업로드합니다. 지원 포맷: **PDF · TXT · DOCX · CSV · MD** (최대 10MB)

    이미 저장된 지식베이스가 있으면 체크박스로 선택할 수 있습니다.

    <Frame caption="지식베이스 연결 — 업로드 또는 기존 선택">
      <img src="/images/지식베이스 선택.png" alt="지식베이스 연결 화면" />
    </Frame>
  </Step>

  <Step title="N개 연결">
    선택한 수만큼 **[N개 연결]** 버튼을 클릭하면 KNOWLEDGE 블록에 파일이 추가됩니다.
  </Step>
</Steps>

### 4-2. TOOLBOX — MCP 도구 선택

<Steps>
  <Step title="+ Add 클릭">
    TOOLBOX 블록의 **`+ Add`** 버튼을 클릭합니다.
  </Step>

  <Step title="도구 검색·선택">
    **[MCP 도구 선택]** 모달에서 EXTERNAL 59종 도구 중 필요한 것을 체크박스로 선택합니다.

    <Frame caption="MCP 도구 선택 — EXTERNAL 59종 중 선택">
      <img src="/images/툴박스.png" alt="MCP 도구 선택 모달" />
    </Frame>
  </Step>

  <Step title="도구를 선택하세요">
    체크 후 **[도구를 선택하세요]** 버튼을 클릭하면 TOOLBOX에 추가됩니다.
  </Step>
</Steps>

<Note>
Slack · Gmail · Google Drive · Notion · GitHub 등 외부 서비스 도구는 [Integrations](#7-integrations--외부-서비스-oauth-연결) 에서 OAuth 계정 연결이 선행되어야 정상 동작합니다.
</Note>

<Steps>

    <Frame caption="Few-shot 예시 추가 — 저장된 예시 라이브러리">

  </Step>

<Tip>
Few-shot 예시는 에이전트의 **톤 · 출력 형식**을 학습시키는 핵심 장치입니다. 서명 · 리포트 양식 · 응답 스타일을 일관화하고 싶다면 반드시 추가하세요.

**⚠️ 주의**: 예시에 `[대괄호]` 플레이스홀더가 남아 있으면 AI가 그 패턴을 그대로 학습해 실제 출력에도 대괄호가 나타납니다. 반드시 **실제 값으로 치환된 완성본**을 사용하세요.
</Tip>

### 4-6. SCHEDULE — 정기 실행 스케줄

<Steps>
  <Step title="+ Add 또는 설정 클릭">

    - **타임존**: 기본 `Asia/Seoul (KST)`

  <Step title="생성">
    **[생성]** 버튼을 클릭하면 SCHEDULE 블록에 등록됩니다.
  </Step>
</Steps>

## 5. AI 수정 도우미로 에이전트 개선

기존의 **워크플로우 검증 → AI 품질 분석 100점 → 심층 분석 → 수정 요청** 플로우가 **AI 수정 도우미 탭 하나로 통합**되었습니다.

<Steps>
  <Step title="AI 수정 도우미 탭 선택">
    좌측 패널에서 **[AI 수정 도우미]** 탭을 클릭합니다.
  </Step>

  <Step title="수정 요청 입력">
    하단 입력창에 수정하고 싶은 내용을 자연어로 입력합니다.

    **예시:**
    - "응답을 더 간결하게, 200단어 이내로 해줘"
    - "서명은 KNOWLEDGE 파일의 B 섹션 값만 사용하도록 해줘"
    - "영역별 아이콘을 추가해서 가독성을 높여줘"
    - "법률 자문 요청은 법무팀 연결로 유도하도록 해줘"
  </Step>

  <Step title="자동 반영 확인">
    AI가 시스템 프롬프트 · 연결된 블록을 자동으로 수정합니다. **에이전트 대화** 탭으로 전환해 변경 결과를 즉시 테스트할 수 있습니다.
  </Step>
</Steps>

<Tip>
**에이전트 대화** 와 **AI 수정 도우미** 를 번갈아 가며 **테스트 → 수정 → 재테스트** 루프를 반복하면 에이전트 품질이 빠르게 올라갑니다.
</Tip>

## 6. 스킬(Skill) 만들기

스킬은 **모든 에이전트에서 공유되는 역할 모듈**입니다. 상단 [Skills] 탭에서 전역 관리합니다.

### 에이전트 vs 스킬

| 구분 | 에이전트 | 스킬 |
|------|----------|------|
| **정의** | 완성된 AI 도우미 단위 | 여러 에이전트에서 재사용되는 역할 부품 |
| **관리 위치** | 상단 [에이전트] 탭 | 상단 [Skills] 탭 |
| **사용 방식** | CHANNELS · SCHEDULE로 직접 실행 | 에이전트의 SKILLS 블록에 연결되어 호출 |

### skill.md 구성 요소

- **name / description**: 스킬 이름과 설명
- **trigger**: 활성화 키워드
- **model / temperature**: 사용 모델과 창의성 수준
- **역할 및 지시사항**: 스킬이 수행할 구체적 작업 내용

### Skills 탭 — 스킬 관리 화면

<Frame caption="Skills 탭 — 모든 에이전트에서 공유되는 스킬 라이브러리">
  <img src="/images/Skills목록.png" alt="Skills 목록 화면" />
</Frame>

- **스킬 검색**: 이름으로 검색, 최신순 · 이름순 정렬
- **+ 스킬 생성**: 우측 상단 버튼으로 새 스킬 생성
- **각 스킬 카드**: 편집 / 삭제 아이콘

### 스킬 생성 — AI 자동 생성

<Steps>
  <Step title="+ 스킬 생성">
    Skills 탭 우측 상단의 **[+ 스킬 생성]** 버튼을 클릭합니다.
  </Step>

  <Step title="자연어 설명 입력">
    원하는 스킬 기능을 설명합니다.

    예: `긴 문서를 핵심만 요약해주는 도우미`
  </Step>

  <Step title="AI 자동 작성 확인">
    AI가 skill.md 파일을 자동 작성하고 미리보기로 보여줍니다.

    <Frame caption="스킬 생성 — AI 자동 생성 결과 미리보기">
      <img src="/images/스킬 생성.png" alt="스킬 생성 결과" />
    </Frame>
  </Step>

  <Step title="저장">
    내용을 확인한 후 **저장**하면 Skills 라이브러리에 등록됩니다. 이후 모든 에이전트의 SKILLS 블록에서 선택 가능합니다.
  </Step>
</Steps>

<Tip>
**더 좋은 스킬을 생성하려면:**
- 입력 내용을 **구체적으로 설명**하세요 (단순 "요약 도우미" 대신 "PDF 논문을 500자 이내 한국어로 요약하는 도우미")
- 원하는 **출력 형식을 명시**하세요 (마크다운·JSON·표 등)
- **특별 규칙·제약**을 포함하세요 (금지 표현·출력 길이·톤)
- **구체적 예시**를 제공하면 품질이 크게 향상됩니다
</Tip>

### 스킬 활용 패턴

| 패턴 | 설명 | 예시 |
|------|------|------|
| **단일 스킬 실행** | 하나의 전문 역할 수행 | 문서 요약 스킬만 연결 |
| **스킬 체이닝** | SKILLS 블록에 여러 스킬 연결 → 상황별 순차 호출 | 번역 → 교정 → 설명 스킬 순차 활용 |
| **서브 에이전트 위임** | 스킬로 감당하기 어려운 큰 단위 작업은 **SUB-AGENTS 블록**으로 별도 에이전트 위임 | 메인 에이전트 → 전문 분석 에이전트 위임 |

### 스킬 라이브러리 활용법

| 기능 | 설명 |
|------|------|
| **검색 & 정렬** | Skills 탭에서 이름으로 검색, 최신순·이름순 정렬 |
| **바로 적용** | 에이전트 SKILLS 블록의 `+ Add` → 체크박스 선택 → 적용 |
| **수정 & 관리** | 저장된 skill.md를 언제든 편집·삭제 — 연결된 모든 에이전트에 일괄 반영 |
| **복제 활용** | 기존 스킬을 복제 → 약간 수정 → 새 스킬로 빠르게 생성 |

### 3가지 핵심 가치

1. **재사용성** — 여러 에이전트에서 활용
2. **일관성** — 균일한 품질 보장
3. **유지보수** — 수정 시 일괄 반영

<Note>
자주 쓰는 역할(요약 · 교정 · 번역 · 이메일 작성 등)은 스킬로 **미리 만들어두세요.** 매번 시스템 프롬프트를 복붙하는 것보다 훨씬 유지보수가 쉽습니다.
</Note>

상단 [Integrations] 탭에서 외부 서비스를 **OAuth 2.0** 으로 연결합니다. 설정 > 외부 연결 경로에서도 접근 가능합니다.

### 연결 가능한 서비스

| 카테고리 | 서비스 | 용도 |
|---------|--------|------|
| **커뮤니케이션** | Gmail | 이메일 발송(본인 전용) |
| **커뮤니케이션** | Slack | 채널 메시지 · DM · 알림 수신 |
| **생산성 도구** | Notion | 페이지 · 데이터베이스 가져오기 |
| **생산성 도구** | Google Drive | 드라이브 파일 읽기 · 학습 자료 가져오기 |
| **개발 도구** | GitHub | 코드 저장소 · PR · Gist 접근 |

<Warning>
**보안 안내** — 연결 시 발급되는 토큰은 서버에서 **암호화**되어 저장되며, 프론트엔드에 노출되지 않습니다. 서비스를 연결 해제하면 저장된 토큰은 비활성화됩니다. Google(Drive/Gmail) 토큰은 서버에서 **자동 갱신**되므로 재연결이 필요 없습니다.
</Warning>

## 8. 내 에이전트 관리

상단 [에이전트] 탭에서 생성한 에이전트를 조회 · 수정 · 공유합니다.

### 에이전트 카드 정보

- 이름 / 설명
- 상태 (활성 / 임시저장)
- 수정일 / 모델명 / 연결된 블록 요약

### 관리 기능

| 기능 | 설명 |
|------|------|
| **대화** | 에이전트와 즉시 대화 시작 |
| **편집** | 에이전트 편집 화면(Workflow 뷰)으로 이동 |
| **공유하기** | 수업에 에이전트 공유 |
| **공유 중지** | 공유 해제 |
| **비활성화** | 에이전트 일시 중지 |
| **삭제** | 에이전트 영구 삭제 |
