# Step 5: 에이전트 빌더 실습

## AI 에이전트 생성

원하는 에이전트를 **자연어로 설명**하면, AI가 최적의 워크플로우를 자동으로 설계합니다.

<Steps>
  <Step title="AI 생성 탭 선택">
    에이전트 빌더 메뉴에서 **[AI 생성]** 탭을 클릭합니다.
  </Step>

  <Step title="원하는 에이전트 설명">
    "만들고 싶은 에이전트를 설명하세요" 입력란에 원하는 기능을 **자연어로 설명**합니다. 하단의 예시(교과서 Q&A 챗봇, 과목별 학습 도우미, 안전한 학습 봇, DART 재무 분석기)를 참고할 수 있습니다.

    <Frame caption="AI 에이전트 생성 — 자연어 입력 화면">
      <img src="/images/에이전트빌더_AI 생성.png" alt="AI 에이전트 생성 입력 화면" />
    </Frame>
  </Step>

  <Step title="AI로 에이전트 생성">
    **[생성하기]** 버튼을 클릭하면 AI가 자동으로 노드를 구성하여 워크플로우를 생성합니다. 좌측에서 생성된 노드 구성과 모델 정보를 확인할 수 있으며, **[JSON]** 버튼을 클릭하면 워크플로우 구조를 JSON 형식으로 확인하고 복사할 수 있습니다.

    <Frame caption="AI 에이전트 생성 — Workflow JSON">
      <img src="/images/에이전트빌더_AI 생성(JSON).png" alt="워크플로우 JSON 확인" />
    </Frame>
  </Step>

  <Step title="빌더에서 열기">
    **[빌더에서 열기]** 버튼을 클릭하면 **[나만의 에이전트 만들기]** 탭의 워크플로우 편집 화면으로 이동합니다. 캔버스에 배치된 노드를 확인하고, 우측 속성 패널에서 각 노드의 시스템 프롬프트와 설정을 수정할 수 있습니다.

    <Frame caption="AI 에이전트 생성 — 빌더에서 열기">
      <img src="/images/에이전트빌더_AI 생성(노드 생성).png" alt="AI가 생성한 워크플로우 편집 화면" />
    </Frame>
  </Step>
</Steps>

<Tip>
**[재생성]** 버튼으로 동일한 설명에 대해 다른 워크플로우 구성을 생성할 수 있습니다.
</Tip>

## 나만의 에이전트 만들기

<Steps>
  <Step title="노드 추가">
    좌측 팔레트에서 원하는 노드를 **드래그하여 캔버스에 배치**합니다.
  </Step>

  <Step title="노드 연결">
    각 노드의 포트를 **화살표로 연결**하여 흐름을 구성합니다. 노드 하단 출력 포트 → 다른 노드 상단 입력 포트로 연결합니다.
  </Step>

  <Step title="속성 설정">
    우측 속성 패널에서 각 노드에 맞게 **설정을 조정**합니다.
  </Step>

  <Step title="워크플로우 검증">
    우측 미리보기 패널에서 **실행 결과를 즉시 확인**합니다. 실행 경로와 노드별 소요시간을 확인할 수 있습니다.

    <Warning>
    검증 실패 시 에이전트는 저장되지 않습니다.
    </Warning>
  </Step>

  <Step title="저장/게시">
    최종 동작 확인 후 **버전 저장**하고 에이전트를 게시합니다. 게시 범위는 **나만 사용** 또는 **수업에 공유** 중 선택할 수 있습니다.
  </Step>
</Steps>

## 스킬(Skill) 개념

스킬은 **특정 역할과 규칙을 정의하여 재사용 가능한 AI 역할 모듈**입니다.

### 에이전트 vs 스킬

| 구분 | 에이전트 | 스킬 |
|------|----------|------|
| **정의** | 완성된 AI 도우미 그 자체 | 에이전트 내부에 포함되는 역할 부품 |
| **사용 방식** | 시스템 프롬프트로 정의하고 직접 대화 | 여러 워크플로우에서 재사용 가능 |

### skill.md 구성 요소

- **name / description**: 스킬 이름과 설명
- **trigger**: 활성화 키워드
- **model / temperature**: 사용 모델과 창의성 수준
- **역할 및 지시사항**: 스킬이 수행할 구체적 작업 내용

### 3가지 핵심 가치

1. **재사용성** — 여러 에이전트에서 활용
2. **일관성** — 균일한 품질 보장
3. **유지보수** — 수정 시 일괄 반영

---

## 스킬 만들기 — AI 도움받기

<Steps>
  <Step title="AI 도움받기 선택">
    에이전트 빌더 > 스킬 노드 클릭 > **"AI 도움받기"** 탭을 선택합니다.
  </Step>

  <Step title="자연어 설명 입력">
    원하는 스킬 기능을 설명합니다.

    예: "긴 문서를 핵심만 요약해주는 도우미"
  </Step>

  <Step title="스킬 자동 생성">
    **"스킬 생성"**을 클릭하면 AI가 skill.md 파일을 자동으로 작성합니다.

    ```yaml
    name: 핵심문서요약
    description: 긴 문서의 핵심 내용을 요약
    trigger: "요약", "핵심", "문서요약"
    model: gpt-4.1-mini
    temperature: 0.2
    ```
  </Step>

  <Step title="품질 향상 및 저장">
    **"예시 추가"** / **"개선하기"**로 품질을 높인 후 저장합니다.
  </Step>
</Steps>

<Tip>
더 좋은 스킬을 생성하려면: 입력 내용을 **구체적으로 설명**하고, 원하는 **출력 형식을 명시**하고, **특별 규칙/제약을 포함**하고, **구체적 예시를 제공**하세요.
</Tip>

## 스킬 저장 & 라이브러리

<Steps>
  <Step title="내 스킬에 저장">
    skill.md를 확인한 후 **"내 스킬에 저장"**을 클릭합니다.
  </Step>

  <Step title="이름 & 설명 입력">
    알아보기 쉬운 **이름을 지정**합니다.
  </Step>

  <Step title="저장하기 완료">
    **내 스킬 라이브러리**에 보관됩니다.
  </Step>
</Steps>

### 라이브러리 활용법

| 기능 | 설명 |
|------|------|
| **검색 & 정렬** | 스킬 이름으로 검색, 최신순/이름순 정렬 |
| **바로 적용** | 체크박스 선택 → "N개 스킬 적용" 클릭 |
| **수정 & 관리** | 저장된 skill.md를 언제든 편집 |
| **복제 활용** | 기존 스킬 복제 → 약간 수정 → 새 스킬 빠르게 생성 |

<Note>
자주 쓰는 역할(요약, 교정, 번역 등)은 **미리 만들어두세요!**
</Note>

## 스킬 활용 패턴

| 패턴 | 설명 | 예시 |
|------|------|------|
| **단일 스킬 실행** | 하나의 전문 역할 수행 | 문서 요약 |
| **스킬 체이닝** | 여러 스킬 순차 연결 | 번역 → 교정 → 설명 |
| **서브 에이전트** | 에이전트가 에이전트 호출 | 메인 → 전문가 |

## 테스트 & 실행

### 워크플로우 검증

에이전트를 저장하기 전에 **워크플로우 검증**을 통해 각 노드가 정상적으로 작동하는지 확인합니다.

<Steps>
  <Step title="워크플로우 검증">
    우측 패널에서 **[워크플로우 검증]** 버튼을 클릭하면 각 노드가 순서대로 실행됩니다. 노드별 상태(완료/검사 중/대기)를 확인할 수 있으며, 모든 노드가 **완료**되면 **검증 완료** 상태가 됩니다.

    <Frame caption="워크플로우 검증 — 노드별 완료 상태 확인">
      <img src="/images/에이전트빌더_(워크플로우 검증).png" alt="워크플로우 검증 화면" />
    </Frame>
  </Step>

  <Step title="AI 품질 분석">
    검증 완료 후 **[AI 품질 분석]**을 실행하면 100점 만점으로 에이전트 품질을 평가합니다. **[기본 분석]**과 **[심층 분석]** 두 가지 모드를 제공합니다.

    <Frame caption="AI 품질 분석 — 100점 만점 평가">
      <img src="/images/에이전트빌더_(AI 품질 분석).png" alt="AI 품질 분석 결과" />
    </Frame>
  </Step>

  <Step title="심층 분석으로 개선점 확인">
    **[심층 분석]**을 클릭하면 경고와 정보 수준의 개선점을 구체적으로 제시합니다. 각 개선점에는 문제 설명과 해결 방안이 포함됩니다.

    <Frame caption="심층 분석 — 개선점 상세 확인">
      <img src="/images/에이전트빌더_(심층 분석).png" alt="심층 분석 결과" />
    </Frame>
  </Step>

  <Step title="수정 요청">
    하단의 **[수정 요청]** 탭을 선택한 후 개선할 내용을 자연어로 입력하면, AI가 워크플로우 자체를 자동으로 수정합니다.

    <Frame caption="수정 요청 — 자연어로 워크플로우 개선">
      <img src="/images/에이전트빌더_(수정 요청).png" alt="수정 요청 화면" />
    </Frame>
  </Step>

  <Step title="수정 완료 & 재검증">
    AI가 워크플로우를 수정하면 **"워크플로우 수정 완료"** 알림이 표시됩니다. 수정된 내용을 확인하고 **재검증**하여 품질 점수 변화를 확인할 수 있습니다.

    <Frame caption="수정 완료 — 워크플로우 자동 개선 결과">
      <img src="/images/에이전트빌더_(수정 완료).png" alt="수정 완료 화면" />
    </Frame>
  </Step>
</Steps>

<Tip>
**테스트 실행**과 **수정 요청**을 반복하며 에이전트 품질을 점진적으로 높일 수 있습니다.
</Tip>

## 내 에이전트 관리

### 탭 구성

- **내 에이전트 탭**: 생성한 에이전트 전체 목록
- **수업 공유 탭**: 훈련생과 공유된 에이전트
- **검색 & 필터**, **+ 새 에이전트**

### 에이전트 카드 정보

- 이름/설명
- 상태 (활성화 / 임시저장)
- 수정일 / 노드 수 / 스킬 태그

### 관리 기능

| 기능 | 설명 |
|------|------|
| **대화** | 에이전트와 대화 시작 |
| **노드편집** | 워크플로우 편집 화면으로 이동 |
| **공유하기** | 수업에 에이전트 공유 |
| **공유 중지** | 공유 해제 |
| **비활성화** | 에이전트 일시 중지 |
| **삭제** | 에이전트 영구 삭제 |
