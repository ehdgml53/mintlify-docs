# Step 2: 프롬프트 엔지니어링

## 기능 소개

프롬프트 엔지니어링은 **AI에게 효과적으로 지시하는 기술**입니다. 잘 작성된 프롬프트는 같은 모델에서도 훨씬 더 좋은 결과를 얻을 수 있게 해줍니다.

<Note>
**AI 자동 생성 기능** — 복잡한 프롬프트 작성법을 몰라도 원하는 역할을 자연어로 설명하면 AI가 시스템 프롬프트와 Few-shot 예시를 자동으로 생성해줍니다.
</Note>

---

## AI 자동 생성 기능

프롬프트와 Few-shot을 직접 작성하지 않아도, **자연어로 원하는 역할을 설명**하면 AI가 자동으로 만들어줍니다.

<Tabs>
  <Tab title="프롬프트 자동 생성">
    원하는 역할을 설명하면 AI가 **시스템 프롬프트를 자동으로 생성**합니다.

    <img src="/images/프롬프트 엔지니어링_AI생성(프롬프트).png" alt="AI 프롬프트 생성 화면" />

    <Steps>
      <Step title="AI 생성 탭 선택">
        프롬프트 메뉴 상단에서 **[AI 생성]** 탭을 클릭합니다.
      </Step>
      <Step title="역할 설명 입력">
        **역할 설명** 입력란에 원하는 역할을 자연어로 설명합니다.

        예: "수학 개념을 단계별로 가르치는 친절한 튜터"
      </Step>
      <Step title="생성하기 클릭">
        **[생성하기]** 버튼을 클릭하면 AI가 전문적인 시스템 프롬프트를 자동 생성합니다.
      </Step>
    </Steps>

    하단의 **예시 카드**(수학 튜터, 영어 번역가, 코드 리뷰어, 요약 전문가)를 클릭하면 역할 설명이 자동 입력됩니다.
  </Tab>

  <Tab title="Few-shot 자동 생성">
    에이전트 역할을 설명하면 AI가 **대화 예시(Few-shot)를 자동으로 생성**합니다.

    <img src="/images/프롬프트 엔지니어링_AI생성(Few-shot).png" alt="AI Few-shot 생성 화면" />

    <Steps>
      <Step title="Few-shot 생성 탭 선택">
        AI 생성 화면 상단에서 **[Few-shot 생성]** 탭을 클릭합니다.
      </Step>
      <Step title="에이전트 역할 입력">
        **에이전트 역할** 입력란에 원하는 역할을 설명합니다.

        예: "수학을 가르치는 친절한 튜터"
      </Step>
      <Step title="생성 개수 선택">
        생성할 예시 개수를 **1~5개** 중 선택합니다. (권장: 2~3개)
      </Step>
      <Step title="예시 생성 클릭">
        **[예시 생성]** 버튼을 클릭하면 AI가 입출력 예시를 자동 생성합니다.
      </Step>
    </Steps>

    하단의 **예시 카드**(Q&A 튜터, 분류 전문가, 코딩 멘토)를 클릭하면 역할이 자동 입력됩니다.
  </Tab>
</Tabs>

<Tip>
AI가 생성한 프롬프트와 Few-shot은 **LLM 실습에서 바로 연결**하여 활용하거나, **에이전트 빌더**에서 스킬로 사용할 수 있습니다.
</Tip>

## 프롬프트 엔지니어링의 효과

1. **정확성 향상 (Accuracy)** — 의도에 부합하는 정확한 답변을 획득합니다.
2. **일관성 유지 (Consistency)** — 반복적인 작업에서도 동일한 품질을 유지합니다.
3. **재사용성 확보 (Reusability)** — 잘 만든 프롬프트는 템플릿으로 저장하여 활용할 수 있습니다.

## [기본] 프롬프트 작성 기법 3가지

### 1. 명확한 지시문

두루뭉술한 질문 대신 **구체적 조건과 범위**를 명시합니다.

| | 프롬프트 |
|---|---|
| **BEFORE** | "이거 설명해줘" |
| **AFTER** | "이 개념을 초보자도 이해할 수 있게 3문장으로 설명해줘" |

### 2. 역할 부여

AI에게 **전문가 역할**을 지정하면 해당 분야에 맞는 깊이 있는 답변을 얻을 수 있습니다.

> 예: "너는 10년 경력의 마케팅 전문가야. 이 제품의 타겟 고객층을 분석해줘."

### 3. 출력 형식 지정

**표, 목록, JSON** 등 원하는 형식을 명시하면 정리된 결과를 받을 수 있습니다.

> 예: "결과를 마크다운 표로 정리해줘. 열은 항목, 장점, 단점으로 구성해."

<Tip>
**3가지 기법을 조합하면 더욱 강력한 프롬프트가 됩니다.**

예: "마케팅 전문가로서**(역할)**, 이 개념을 초보자용으로 설명하되**(지시문)**, 표로 정리해줘**(형식)**"
</Tip>

## 시스템 프롬프트 (System Prompt)

### 개념

시스템 프롬프트는 대화 전체에 적용되는 **AI의 기본 설정**입니다. 마치 연극의 **'배역 설정'** 과 같아서, AI가 어떤 역할로 어떤 스타일로 응답할지를 정의합니다.

### 4대 구성 요소

<AccordionGroup>
  <Accordion title="1. 역할 (Role)">
    AI가 어떤 전문가인지 **정체성을 정의**합니다.

    > 예: "10년 경력의 시니어 개발자이자 교육 전문가"
  </Accordion>

  <Accordion title="2. 전문성 (Expertise)">
    담당할 수 있는 **지식 영역을 명시**합니다.

    > 예: "Python, 알고리즘, 코드 디버깅 전문"
  </Accordion>

  <Accordion title="3. 톤 & 스타일">
    응답의 **분위기, 말투, 난이도를 설정**합니다.

    > 예: "친근하고 격려하는 톤, 초보자도 이해하기 쉽게"
  </Accordion>

  <Accordion title="4. 제약 조건">
    **하지 말아야 할 것과 지켜야 할 규칙**을 정합니다.

    > 예: "전문 용어 사용 자제, 비유와 예시 필수"
  </Accordion>
</AccordionGroup>

## Few-shot 학습

### 개념

Few-shot은 AI에게 **"이런 식으로 대답해 줘"** 라는 패턴을 예시로 보여주어 답변 방식을 학습시키는 기법입니다. 단순히 말로 설명하는 것보다 **구체적인 입력-출력 예시를 2~3개 제공**했을 때 AI의 성능과 형식 준수율이 크게 향상됩니다.

### 예시 제공량에 따른 분류

| 유형 | 설명 |
|------|------|
| **Zero-shot** | 예시 없이 바로 질문. AI의 기본 지식에 의존 |
| **One-shot** | 하나의 예시를 제공하여 대략적인 방향성 제시 |
| **Few-shot** | 여러 개의 예시를 통해 패턴을 명확히 학습. 가장 안정적이고 품질 높은 결과 |

<Note>
예시가 많을수록 정확도는 높아지지만 **토큰 사용량이 증가**합니다. 일반적으로 2~3개의 예시가 비용 대비 효과가 가장 좋습니다.
</Note>

### Few-shot 사용법

<Tabs>
  <Tab title="직접 작성">
    직접 예시를 작성하여 AI에게 원하는 응답 형식을 가르치는 방법입니다.

    대화 입력 시 아래와 같은 형태로 예시를 포함합니다:

    ```
    [예시 1]
    입력: 사과
    출력: 빨간색의 달콤한 과일로, 비타민이 풍부합니다.

    [예시 2]
    입력: 바나나
    출력: 노란색의 부드러운 과일로, 칼륨이 풍부합니다.

    이제 아래 입력에 대해 같은 형식으로 답해주세요.
    입력: 포도
    ```
  </Tab>

  <Tab title="템플릿 불러오기">
    미리 저장된 Few-shot 템플릿을 불러와서 적용하는 방법입니다.

    <Steps>
      <Step title="템플릿 열기">
        입력 창의 **[+] 버튼**을 클릭하고, '파일 & 문서' 영역에서 **[템플릿 사용]** 을 클릭합니다.
      </Step>
      <Step title="Few-shot 탭 선택">
        템플릿 팝업 상단에서 **[Few-shot]** 탭을 선택합니다.
      </Step>
      <Step title="예시 적용">
        목록에서 원하는 예시를 선택하면 **자동으로 입력창에 적용**됩니다.
      </Step>
    </Steps>
  </Tab>
</Tabs>
