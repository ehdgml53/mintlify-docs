# Step 6: 에이전트 빌더 개요

## 기능 소개

에이전트 빌더는 복잡한 코딩 없이 **노드(Node)를 연결하여 나만의 AI 도구를 시각적으로 설계**하는 워크플로우 도구입니다.

- **노드 기반 워크플로우**: 시작부터 종료까지 각 단계를 블록 쌓듯이 연결
- **데이터 처리 흐름**: 입력 → 프롬프트 → 스킬/MCP/RAG → 결과
- **자율 실행(Autonomous)**: AI가 판단하여 필요한 도구를 스스로 호출

---

## 에이전트 빌더 활용 4단계

<Steps>
  <Step title="개념 이해">
    워크플로우의 **구조와 작동 원리**를 이해합니다.
  </Step>

  <Step title="설계">
    필요한 노드를 추가하고 **논리적으로 연결**합니다.
  </Step>

  <Step title="테스트">
    워크플로우를 실행하고 **오류를 수정(디버깅)**합니다.
  </Step>

  <Step title="배포 & 공유">
    완성된 에이전트를 **해당 클래스에 게시**하고 활용합니다.
  </Step>
</Steps>

## 기본 워크플로우 노드 6종

<CardGroup cols={2}>
  <Card title="시작 노드 (Start)" icon="play">
    워크플로우의 **시작점**. 사용자의 초기 입력 메시지를 받아 다음 단계로 전달합니다. **필수 노드**입니다.
  </Card>

  <Card title="종료 노드 (End)" icon="flag-checkered">
    워크플로우 실행을 마치고 **최종 결과를 사용자에게 출력**합니다. 모든 흐름은 이 노드에서 끝납니다.
  </Card>

  <Card title="프롬프트 노드 (Prompt)" icon="message">
    LLM에게 **구체적인 지시**를 전달합니다. 역할(Persona) 부여, 텍스트 생성 등에 사용합니다.
  </Card>

  <Card title="스킬 노드 (Skill)" icon="star">
    **특정 기능을 수행**하는 도구입니다. 퀴즈 생성, 문법 교정, 학습 플래너 등 미리 정의된 기능을 실행합니다.
  </Card>

  <Card title="지식베이스 노드 (Knowledge)" icon="database">
    **RAG 기술을 활용**합니다. 업로드된 문서에서 질문 관련 내용을 검색/참조합니다.
  </Card>

  <Card title="MCP 도구 노드 (MCP)" icon="plug">
    **외부 서비스를 연동**합니다. 실시간 웹 검색, 계산, 환율 조회 등을 수행합니다.
  </Card>
</CardGroup>

## 로직 & 고급 노드 5종

<CardGroup cols={2}>
  <Card title="분류 노드 (Classification)" icon="tags">
    사용자의 입력 의도를 분석하여 **적절한 경로로 자동 분기**합니다.

    예: "문의 내용이 환불/배송/기타 중 어디에 해당하는지 자동 분류"
  </Card>

  <Card title="조건분기 노드 (If/Else)" icon="code-branch">
    변수 값이나 특정 조건의 충족 여부에 따라 **흐름을 제어**합니다.

    예: "회원 등급이 VIP인 경우 전담 상담원 봇으로 연결"
  </Card>

  <Card title="변환 노드 (Transform)" icon="arrows-spin">
    데이터 포맷 변경이나 **특정 값 추출/가공**을 수행합니다.

    예: "JSON 응답을 텍스트 리포트로 변환"
  </Card>

  <Card title="상태설정 노드 (Set State)" icon="floppy-disk">
    대화 문맥이나 사용자 정보를 **변수에 저장/업데이트**합니다.

    예: "언어 설정이나 관심사 정보를 기억하여 다음 대화에 반영"
  </Card>

  <Card title="가드레일 노드 (Guardrail)" icon="shield-halved">
    입출력 데이터를 검사하고 **부적절한 내용을 필터링/보안 강화**합니다.

    예: "응답에 개인정보 포함 시 마스킹 처리"
  </Card>
</CardGroup>

## 멀티모달 노드 3종

<Note>
멀티모달 노드는 현재 **준비중**입니다. 출시 시 아래 기능이 제공될 예정입니다.
</Note>

<CardGroup cols={3}>
  <Card title="비전 분석 노드" icon="eye">
    이미지 심층 분석 → 인사이트/설명 도출
  </Card>

  <Card title="이미지 생성 노드" icon="image">
    텍스트 프롬프트 → 새 이미지 창작
  </Card>

  <Card title="OCR 추출 노드" icon="scanner-image">
    이미지 속 텍스트 인식 → 디지털 데이터 변환
  </Card>
</CardGroup>

<Tip>
비전 분석 노드 뒤에 **'조건 분기' 노드**를 연결하면, 이미지 분석 결과에 따라 다른 처리를 수행할 수 있습니다.
</Tip>

## 에이전트 핵심 개념 4가지

<AccordionGroup>
  <Accordion title="워크플로우">
    노드(블록)와 연결선으로 이루어진 **작업 흐름**입니다. AI가 어떤 순서로 일할지 정하는 설계도입니다.
  </Accordion>

  <Accordion title="노드">
    워크플로우의 **각 단계를 담당하는 블록**입니다. 프롬프트, 분류, 도구 호출 등 다양한 종류가 있습니다.
  </Accordion>

  <Accordion title="스킬">
    **재사용 가능한 AI 능력 모듈**입니다. 한 번 만들면 여러 에이전트에서 반복 사용할 수 있습니다.
  </Accordion>

  <Accordion title="세션">
    에이전트와의 **연속 대화 단위**입니다. 이전 대화 내용을 기억하며 맥락을 이어갑니다.
  </Accordion>
</AccordionGroup>

## MCP 도구 vs 에이전트 비교

| 구분 | MCP 도구 (Step 5) | 에이전트 빌더 (Step 6) |
|------|-------------------|----------------------|
| **역할** | AI에게 쥐여주는 도구 | AI의 행동 흐름을 설계 |
| **비유** | 연필, 계산기, 사전 같은 도구 | 업무 매뉴얼, 순서도 같은 절차 |
| **관계** | 에이전트 안에서 MCP 노드로 사용 | MCP 도구를 워크플로우의 한 단계로 활용 |
