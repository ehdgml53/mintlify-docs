# Step 5: 에이전트 빌더 개요

## 기능 소개

에이전트 빌더는 나 대신 일하는 AI 자동화 에이전트를 **노코드로 설계**하는 도구입니다. 복잡한 노드 연결 대신, **하나의 에이전트 정보 + 8종 구성 블록**에 항목을 추가하는 방식으로 빠르게 에이전트를 완성합니다.

- **단일 에이전트 · 블록 구성 방식**: 이름·설명·프롬프트·모델을 한 곳에서 정의하고, 필요한 능력을 블록에 추가
- **자연어 자동 생성**: "5분 만에 나만의 AI 비서를" — 원하는 에이전트를 설명하면 AI가 자동 구성
- **자율 실행(Autonomous)**: 사용자 요청을 받으면 AI가 연결된 도구·지식베이스·스킬·서브에이전트를 스스로 판단해 호출

<Note>
2026년 4월 UI 개편으로 기존 16종 노드 드래그앤드롭 워크플로우 방식에서 **단일 에이전트 + 구성 블록** 방식으로 변경되었습니다. 워크플로우 검증·AI 품질 분석 기능은 **AI 수정 도우미**로 통합되었습니다.
</Note>

## 에이전트 빌더 활용 4단계

<Steps>
  <Step title="개념 이해">
    에이전트 정보 블록과 **8종 구성 블록의 역할**을 이해합니다.
  </Step>

  <Step title="생성">
    자연어로 설명해 **AI 자동 생성**을 사용하거나, 빈 에이전트에서 블록을 직접 구성합니다.
  </Step>

  <Step title="대화·수정">
    좌측 **에이전트 대화**로 결과를 확인하고, **AI 수정 도우미**에 자연어로 요청해 에이전트를 개선합니다.
  </Step>

  <Step title="배포 & 공유">
    완성된 에이전트를 **수업에 게시**하거나 CHANNELS로 Web·Slack·API·Email 연동합니다.
  </Step>
</Steps>

## 에이전트 구성 — 정보 블록 + 8종 구성 블록

에이전트는 **중앙의 에이전트 정보 블록**과 이를 둘러싼 **8종 구성 블록**으로 이루어집니다. 필요한 블록만 추가하면 되며, 모든 블록은 `+ Add` 버튼으로 항목을 붙입니다.

### 에이전트 정보 (중앙)

<Card title="에이전트 정보 (DESCRIPTION + PROMPT)" icon="id-card">
  에이전트의 **이름·설명·모델·시스템 프롬프트**를 정의합니다. 블록 우측 상단 **`Edit`** 버튼을 클릭하면 **[에이전트 설정]** 모달이 열리며, `프롬프트 불러오기` 기능으로 저장된 프롬프트를 가져올 수 있습니다.

  - **이름**: 에이전트 식별명 (예: "신입사원 온보딩 네비게이터")
  - **설명**: 에이전트 목적 요약 (1~2줄)
  - **모델**: gpt-4.1 등 LLM 모델 선택
  - **시스템 프롬프트**: 에이전트가 따를 역할·규칙·출력 형식 정의
</Card>

## 8종 구성 블록

<CardGroup cols={2}>
  <Card title="KNOWLEDGE" icon="database">
    에이전트가 참조할 **지식베이스** 파일을 연결합니다. 파일을 **드래그하거나 클릭하여 업로드** 하거나(`PDF·TXT·DOCX·CSV·MD`, 최대 10MB), 저장된 지식베이스 중 선택합니다.

    예: 회사 온보딩 정보·응답 정책·FAQ 문서
  </Card>

  <Card title="TOOLBOX" icon="toolbox">
    에이전트가 호출할 **MCP 외부 도구**를 선택합니다. 플랫폼 도구 105개 중 체크박스로 자유 선택.

    예: 뉴스 검색·웹 검색·차트 시각화·Google Docs 문서 생성·DART 전자공시
  </Card>

  <Card title="SKILLS" icon="bolt">
    Skills 탭의 **공유 스킬 라이브러리**에서 필요한 스킬을 연결합니다. 여러 에이전트에서 재사용되는 역할 모듈.

    예: business-email-drafter, weekly-market-report
  </Card>

  <Card title="SUB-AGENTS" icon="sitemap">
    이 에이전트가 **위임할 수 있는 다른 에이전트**를 선택합니다. 복잡한 업무를 여러 에이전트가 분담.

    예: 메인 에이전트 → 이메일 작성 에이전트·경쟁사 분석 에이전트 위임
  </Card>

  <Card title="FEW-SHOT EXAMPLES" icon="comments">
    저장된 **예시 대화(USER/AI 쌍)** 를 선택해 시스템 프롬프트에 자동 추가합니다. 에이전트의 톤·출력 형식을 학습시키는 핵심 장치.

    예: 면접 코치 예시·기획서 피드백 예시
  </Card>

  <Card title="SCHEDULE" icon="clock">
    **정기 실행 스케줄**을 설정합니다. 자연어(`매일 밤 10시`, `매주 월요일 아침 9시`)로 입력하면 Cron 표현식으로 자동 변환. 타임존 선택(기본 Asia/Seoul).

    예: 매일 저녁 상권 리포트 자동 전송
  </Card>

  <Card title="CHANNELS" icon="tower-broadcast">
    에이전트를 외부에서 **호출할 채널**을 설정합니다. 채널 토글과 OAuth 연결 두 탭으로 관리.

    - **채널 토글**: Web(기본) · Slack · API · Email
    - **OAuth 연결**: Google Drive · Gmail · Slack · GitHub · Notion 계정 연동
  </Card>

  <Card title="Folder 뷰" icon="folder">
    우측 상단 **`Folder`** 탭을 클릭하면 Workflow 뷰 대신 **폴더 구조로 에이전트 자산**(지식베이스·스킬·설정 파일 등)을 조회할 수 있습니다.
  </Card>
</CardGroup>

<Tip>
**최소 구성**: 에이전트 정보(PROMPT만) + CHANNELS(Web) 만으로도 작동합니다. KNOWLEDGE·TOOLBOX·SKILLS는 필요 시 추가하세요.
</Tip>

## 좌측 대화 패널 — 2개 탭

<CardGroup cols={2}>
  <Card title="에이전트 대화" icon="comment">
    설정한 에이전트와 **실시간으로 테스트 대화**합니다. 메시지를 입력하면 연결된 KNOWLEDGE·TOOLBOX·SKILLS·SUB-AGENTS를 활용해 답변을 생성합니다.
  </Card>

  <Card title="AI 수정 도우미" icon="pen-to-square">
    **자연어로 에이전트 수정을 요청**합니다. 예: "응답을 더 간결하게 해줘", "서명 규칙을 KNOWLEDGE 파일에서만 가져오도록 해줘" → AI가 PROMPT·블록 설정을 자동 수정합니다.

    기존의 워크플로우 검증·AI 품질 분석·수정 요청 기능이 **이 탭으로 통합**되었습니다.
  </Card>
</CardGroup>

## 에이전트 핵심 개념 5가지

<AccordionGroup>
  <Accordion title="에이전트 (Agent)">
    이름·설명·모델·시스템 프롬프트로 정의되는 **AI 도우미 단위**. 연결된 블록의 능력을 활용해 자율적으로 작업을 수행합니다.
  </Accordion>

  <Accordion title="구성 블록 (Block)">
    에이전트에 **능력을 더해주는 항목**(KNOWLEDGE·TOOLBOX·SKILLS·SUB-AGENTS·FEW-SHOT·SCHEDULE·CHANNELS). 각 블록의 `+ Add` 버튼으로 항목을 추가합니다.
  </Accordion>

  <Accordion title="스킬 (Skill)">
    **모든 에이전트에서 공유되는 재사용 가능한 역할 모듈**. 상단 `Skills` 탭에서 전역 관리하며, 한 번 만들면 여러 에이전트의 SKILLS 블록에서 재사용합니다. 요약·교정·번역 등 공통 기능을 표준화합니다.
  </Accordion>

  <Accordion title="서브에이전트 (Sub-agent)">
    메인 에이전트가 **특정 업무를 위임**하는 에이전트. SUB-AGENTS 블록에서 선택하면, 메인 에이전트가 복잡한 작업 중 해당 영역을 자동으로 위임합니다. 스킬보다 **더 큰 단위의 위임**이 필요할 때 사용합니다.
  </Accordion>

  <Accordion title="세션 (Session)">
    에이전트와의 **연속 대화 단위**. 이전 대화 내용을 기억하며 맥락을 이어가므로, "아까 말한 그 회사"처럼 대명사로 지시해도 이해합니다. 대화 중 연결된 KNOWLEDGE·TOOLBOX 호출 결과도 같은 세션 내에서 참조 가능합니다.
  </Accordion>
</AccordionGroup>

## MCP 도구 vs 에이전트 비교

| 구분 | MCP 도구 (Step 4) | 에이전트 빌더 (Step 5) |
|------|-------------------|----------------------|
| **역할** | AI에게 쥐여주는 도구 | AI의 역할·지시·규칙·데이터 소스를 정의 |
| **비유** | 연필·계산기·사전 같은 도구 | 업무 담당자(역할과 매뉴얼을 가진) |
| **관계** | 에이전트의 **TOOLBOX 블록**에서 선택하여 활용 | MCP 도구를 블록에 추가하고 자율 호출 |
