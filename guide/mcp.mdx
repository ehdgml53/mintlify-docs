# Step 4: MCP 도구 연결

## 기능 소개

MCP(Model Context Protocol)는 **AI 모델이 외부 도구를 호출하는 표준 방식**입니다. AI에게 "손과 발"을 달아주는 기능으로, **70종 이상의 외부 도구**를 제공합니다.

- **핵심 포인트**: AI가 학습된 지식에만 의존하지 않고 실시간 데이터를 조회하거나 도구를 사용하여 더 정확한 답변을 제공합니다.
- **AI 자동 구성 기능**으로 프롬프트 하나만 입력하면 필요한 도구를 자연스럽게 연결해줍니다.

> **프로세스**: 사용자 질문 → AI 모델(LLM) → MCP 표준 프로토콜(중개자) → 외부 도구(날씨 API, 검색 엔진, 계산기 등) → 결과 → 답변

## MCP vs 에이전트 차이

| 구분 | MCP (도구) | 에이전트 (두뇌) |
|------|-----------|----------------|
| **역할** | AI의 손과 발 — 실행 및 데이터 조회 | AI의 두뇌 — 판단 및 지시 |
| **기능** | 외부 시스템/API 연결, 실시간 데이터 확인, 정확한 계산 | 시스템 프롬프트로 페르소나 정의, 작업 단계 계획/추론, 사용자 의도 파악 |
| **비유** | 칼과 불(요리 도구) | 요리사 |

<Note>
요리사(에이전트)가 아무리 뛰어나도 **칼과 불(MCP) 없이는 요리 불가**. 두 가지를 함께 사용할 때 최고의 결과를 얻을 수 있습니다.
</Note>

## MCP ON/OFF 차이

| 질문 | MCP OFF | MCP ON |
|------|---------|--------|
| **현재 시간 확인** | "실시간 정보를 알 수 없어 확인 불가" | "현재 서울 시간은 2026년 2월 26일 오후 4시 30분" |
| **실시간 날씨** | "인터넷 접속 불가로 날씨 확인 불가" | "서울 기온 15°C, 맑음, 강수 확률 0%" |
| **복잡한 수학 계산** | "약 9천 7백만 정도" (부정확) | "정확히 97,406,784" (Calculator 도구 활용) |
| **실시간 환율** | "2023년까지 지식으로 정확한 환율 불가" | "원/달러 환율 1,450.50원 (전일 대비 ▲5.0원)" |

## 7가지 MCP 템플릿

| 템플릿 | 설명 | 권장 도구 |
|--------|------|----------|
| **학습 도우미** | 수업 내용 이해/학습 보조 | 웹 검색, 계산기, 날짜/시간 |
| **리서치** | 정보 수집과 분석 | 웹 검색, 웹 크롤링, 파일 읽기 |
| **코딩 연습** | 프로그래밍 학습 | 계산기, 웹 검색, 날짜/시간 |
| **글쓰기** | 작문/에세이 작성 | 웹 검색, 파일 읽기 |
| **수학/과학** | 수학 계산/과학 탐구 | 계산기, 웹 검색 |
| **외국어 학습** | 언어 학습/번역 | 웹 검색, 웹 크롤링 |
| **자유 구성** | 나만의 MCP 구성 | 모든 도구 선택 가능 |

## MCP 실행 4단계 프로세스

<Steps>
  <Step title="프롬프트 전송">
    사용자 메시지와 연결된 도구 목록이 AI에게 **함께 전달**됩니다.
  </Step>

  <Step title="도구 선택 판단">
    AI가 요청을 분석하여 **필요한 도구를 자율적으로 결정**합니다.
  </Step>

  <Step title="도구 실행">
    선택된 도구가 **외부 서버에서 실제로 동작**합니다.
  </Step>

  <Step title="응답 생성">
    도구 실행 결과를 AI가 받아 **자연어 답변으로 가공**합니다.
  </Step>
</Steps>

## MCP 만들기 (외부 도구)

<Steps>
  <Step title="목적 설정">
    어떤 작업에 사용할지 **명확한 목적**을 선택합니다.
  </Step>

  <Step title="도구 선택">
    플랫폼 도구를 선택합니다.
  </Step>

  <Step title="이름 & 저장">
    MCP 이름, 설명을 작성합니다.
  </Step>

  <Step title="조합 테스트">
    모델 선택과 프롬프트 작성 후 실행합니다.
  </Step>

  <Step title="결과 확인">
    실행 결과를 확인합니다.
  </Step>
</Steps>

<Tip>
결과를 확인한 후 MCP 상태를 **OFF로 변경**하면 MCP가 잘 작동되는지 비교 확인할 수 있습니다.
</Tip>

## 도구 만들기 주요 용어

<AccordionGroup>
  <Accordion title="Input Schema">
    도구가 받는 **입력값을 정의하는 설명서**입니다. 비주얼 모드(폼 입력) 또는 JSON 모드(직접 코드 작성)를 제공합니다.
  </Accordion>

  <Accordion title="Handler Code">
    입력값을 받아 **실제로 실행되는 JavaScript 코드**입니다. AI 도움받기로 자동 작성이 가능합니다.
  </Accordion>
</AccordionGroup>

<Note>
코딩 경험이 없어도 괜찮습니다. 생성된 코드가 이해되지 않아도 **저장 후 테스트에서 잘 작동하면** 그대로 사용하면 됩니다.
</Note>

## 내 MCP 목록 관리

생성된 MCP는 다음과 같이 관리할 수 있습니다.

| 기능 | 설명 |
|------|------|
| **활성/비활성 토글** | ON/OFF로 MCP를 켜고 끕니다 |
| **테스트 실행** | MCP가 정상 작동하는지 확인합니다 |
| **수정 및 편집** | 설정을 변경합니다 |
| **공유 및 가져오기** | 팀원들과 MCP 구성을 공유합니다 (준비중) |
| **삭제** | 불필요한 MCP를 제거합니다 |
