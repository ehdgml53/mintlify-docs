### 7가지 MCP 템플릿

| 템플릿 | 설명 | 권장 도구 |
|--------|------|----------|
| 학습 도우미 | 수업 내용 이해/학습 보조 | 웹 검색, 계산기, 날짜/시간 |
| 리서치 | 정보 수집과 분석 | 웹 검색, 웹 크롤링, 파일 읽기 |
| 코딩 연습 | 프로그래밍 학습 | 계산기, 웹 검색, 날짜/시간 |
| 글쓰기 | 작문/에세이 작성 | 웹 검색, 파일 읽기 |
| 수학/과학 | 수학 계산/과학 탐구 | 계산기, 웹 검색 |
| 외국어 학습 | 언어 학습/번역 | 웹 검색, 웹 크롤링 |
| 자유 구성 | 나만의 MCP 구성 | 모든 도구 선택 가능 |

### MCP 실행 4단계 프로세스

<Steps>
  <Step title="프롬프트 전송">
    사용자 메시지와 도구 목록이 AI에게 전달됩니다.
  </Step>
  <Step title="도구 선택 판단">
    AI가 요청을 분석하여 필요한 도구를 자율적으로 결정합니다.
  </Step>
  <Step title="도구 실행">
    선택된 도구가 외부 서버에서 동작합니다.
  </Step>
  <Step title="응답 생성">
    도구 결과를 AI가 자연어 답변으로 가공하여 사용자에게 전달합니다.
  </Step>
</Steps>

### MCP 만들기

<Steps>
  <Step title="목적 설정">
    어떤 작업에 사용할지 명확한 목적을 선택합니다.
  </Step>
  <Step title="도구 선택">
    플랫폼에서 제공하는 도구 중 필요한 것을 선택합니다.
  </Step>
  <Step title="이름 & 저장">
    MCP 이름과 설명을 작성합니다.
  </Step>
  <Step title="조합 테스트">
    모델을 선택하고 프롬프트를 작성한 후 실행하여 테스트합니다.
  </Step>
  <Step title="결과 확인">
    도구 호출 결과와 AI 응답을 확인합니다.
  </Step>
</Steps>

<Tip>
  MCP 상태를 **OFF**로 변경 후 비교하면 MCP가 잘 작동되는지 확인할 수 있습니다.
</Tip>

## 에이전트 빌더

### 에이전트 빌더 활용 4단계

<Steps>
  <Step title="개념 이해">
    워크플로우의 구조와 작동 원리를 이해합니다.
  </Step>
  <Step title="설계">
    필요한 노드를 추가하고 논리적으로 연결합니다.
  </Step>
  <Step title="테스트">
    워크플로우를 실행하고 오류를 수정(디버깅)합니다.
  </Step>
  <Step title="배포·공유">
    완성된 에이전트를 해당 클래스에 게시하고 활용합니다.
  </Step>
</Steps>

### 에이전트 핵심 개념 4가지

<Accordion title="워크플로우 (Workflow)">
  에이전트의 전체 동작 흐름을 정의하는 설계도입니다. 노드와 연결선으로 구성되며, 시작부터 종료까지의 처리 과정을 시각적으로 표현합니다.
</Accordion>

<Accordion title="노드 (Node)">
  워크플로우를 구성하는 개별 작업 단위입니다. 각 노드는 특정 기능(프롬프트 실행, 도구 호출, 조건 분기 등)을 수행합니다.
</Accordion>

<Accordion title="스킬 (Skill)">
  재사용 가능한 프롬프트 + 설정 조합입니다. 자주 사용하는 작업을 스킬로 저장하여 워크플로우에서 반복 활용할 수 있습니다.
</Accordion>

<Accordion title="세션 (Session)">
  에이전트와 사용자 간의 대화 단위입니다. 하나의 세션 내에서 컨텍스트가 유지되며, 워크플로우가 실행됩니다.
</Accordion>

### MCP 도구 vs 에이전트 비교

| 구분 | MCP 도구 | 에이전트 빌더 |
|------|---------|-------------|
| 역할 | AI에게 쥐여주는 도구 | AI의 행동 흐름을 설계 |
| 비유 | 연필, 계산기, 사전 | 업무 매뉴얼, 순서도 |
| 관계 | 에이전트 안에서 MCP 노드로 사용 | MCP 도구를 워크플로우의 한 단계로 활용 |

### 스킬 만들기 -- AI 도움받기

<Steps>
  <Step title="AI 도움받기 선택">
    스킬 생성 화면에서 **AI 도움받기** 옵션을 선택합니다.
  </Step>
  <Step title="자연어 설명 입력">
    만들고 싶은 스킬의 목적과 동작을 자연어로 설명합니다.
  </Step>
  <Step title="스킬 자동 생성">
    AI가 설명을 바탕으로 스킬 구성을 자동 생성합니다.
  </Step>
  <Step title="품질 향상 및 저장">
    생성된 스킬을 검토하고, 필요 시 수정한 뒤 저장합니다.
  </Step>
</Steps>

#### 스킬 예시

```markdown
## 역할
당신은 전문 마케팅 카피라이터입니다.

## 지시사항
- 사용자가 제공한 제품 정보를 바탕으로 광고 카피를 작성합니다.
- 톤: 친근하고 설득력 있는 어조
- 길이: 3문장 이내

## 출력 형식
1. 헤드라인
2. 서브 카피
3. CTA (Call to Action)
```

### 스킬 저장 & 라이브러리

<Steps>
  <Step title="'내 스킬에 저장' 클릭">
    스킬 편집 화면에서 **내 스킬에 저장** 버튼을 클릭합니다.
  </Step>
  <Step title="이름 & 설명 입력">
    스킬의 이름과 설명을 입력합니다.
  </Step>
  <Step title="'저장하기' 완료">
    저장 버튼을 클릭하여 라이브러리에 등록합니다.
  </Step>
</Steps>

#### 라이브러리 활용

- **검색/정렬** -- 저장된 스킬을 검색하거나 정렬하여 빠르게 찾기
- **바로 적용** -- 라이브러리에서 선택하여 워크플로우에 즉시 적용
- **수정/관리** -- 기존 스킬의 내용을 수정하고 관리
- **복제 활용** -- 기존 스킬을 복제하여 변형 버전 생성

### 워크플로우 설계 핵심 단계

<Steps>
  <Step title="노드 추가">
    좌측 팔레트에서 필요한 노드를 드래그하여 캔버스에 배치합니다.
  </Step>
  <Step title="노드 연결">
    각 노드의 포트를 화살표로 연결하여 실행 순서를 정의합니다.
  </Step>
  <Step title="속성 설정">
    우측 속성 패널에서 각 노드의 세부 설정을 구성합니다.
  </Step>
  <Step title="워크플로우 검증">
    미리보기 패널에서 실행 결과를 확인합니다.
  </Step>
  <Step title="저장/게시">
    최종 동작을 확인한 후 저장하고 게시합니다.
  </Step>
</Steps>

---

### 테스트 & 실행

<Steps>
  <Step title="워크플로우 검증">
    전체 워크플로우의 연결 상태와 설정을 검증합니다.
  </Step>
  <Step title="노드별 결과 확인">
    각 노드의 실행 결과를 개별적으로 확인합니다.
  </Step>
  <Step title="오류 디버깅">
    오류가 발생한 노드를 찾아 수정합니다.
  </Step>
  <Step title="게시 및 공유">
    테스트 완료 후 에이전트를 게시하고 학생들에게 공유합니다.
  </Step>
</Steps>

#### 상태 표시등

| 상태 | 설명 |
|------|------|
| 완료 | 노드 실행이 정상적으로 완료됨 |
| 검사 중 | 현재 노드가 실행 중 |
| 대기 | 아직 실행되지 않은 상태 |
| 오류 | 실행 중 오류가 발생함 |

### 내 에이전트 관리

#### 화면 구성

- **내 에이전트 탭** -- 내가 만든 에이전트 목록
- **수업 공유 탭** -- 강의에 공유된 에이전트 목록
- **검색 & 필터** -- 에이전트를 검색하고 필터링
- **+ 새 에이전트** -- 새로운 에이전트 생성

#### 에이전트 카드

| 항목 | 설명 |
|------|------|
| 이름/설명 | 에이전트의 이름과 설명 |
| 상태 | 활성화 또는 임시저장 |
| 수정일 | 마지막으로 수정된 날짜 |

#### 관리 기능

| 기능 | 설명 |
|------|------|
| 대화 | 에이전트와 대화를 시작합니다 |
| 노드편집 | 워크플로우 편집 화면으로 이동합니다 |
| 공유하기 | 특정 강의에 에이전트를 공유합니다 |
| 공유 중지 | 공유를 중지합니다 |
| 비활성화 | 에이전트를 비활성화합니다 |
| 삭제 | 에이전트를 삭제합니다 |

<Warning>
  에이전트를 삭제하면 복구할 수 없습니다. 삭제 전 반드시 확인하세요.
</Warning>
